## 1. 技术栈

| 层级 | 选型 | 原因 |

| 框架 | Next.js 14+ (App Router) | SSR/SSG 支持，文件系统路由，React 生态 |

| 包管理 | pnpm | 速度快，依赖管理干净 |

## 3. 数据管理

### 3.1 设计思路

所有个人数据集中在 `src/data/portfolio.ts` 一个文件中管理，组件只负责渲染，不关心数据来源。这样做的好处：

- **修改方便** — 改数据不需要动组件代码
- **类型安全** — TypeScript 类型约束，防止字段遗漏
- **便于扩展** — 后续接入 CMS 或 Markdown 只需替换数据源

### 3.2 数据结构

```typescript
// src/types/portfolio.ts

interface PersonalInfo {
  name: string           // 姓名
  title: string          // 头衔（如 "AI 工程师"）
  description: string    // 一句话简介
  bio: string            // 详细介绍（支持多段）
  avatar: string         // 头像路径
  resumeUrl?: string     // 简历下载链接
}

interface Experience {
  type: 'work' | 'education'
  company: string        // 公司或学校
  role: string           // 职位或专业
  period: string         // 时间段（如 "2022 - 2024"）
  description: string    // 职责或学习内容
  logo?: string          // 公司/学校 Logo
}

interface Skill {
  category: string       // 分类名（如 "编程语言"）
  items: string[]        // 具体技能列表
}

interface Project {
  title: string          // 项目名
  role: string           // 角色
  description: string    // 描述
  techStack: string[]    // 技术栈
  image?: string         // 截图
  url?: string           // 在线链接
  github?: string        // GitHub 链接
}

interface Contact {
  email: string
  github?: string
  linkedin?: string
  twitter?: string
  wechat?: string
}

interface PortfolioData {
  personalInfo: PersonalInfo
  experiences: Experience[]
  skills: Skill[]
  projects: Project[]
  contact: Contact
}
```

## 4. 主题管理

使用 Tailwind CSS 的 `dark` class 策略实现浅色/深色主题切换：

- 默认跟随系统偏好
- 用户可手动切换，偏好存储在 `localStorage` 中
- 所有颜色使用 Tailwind 的 `dark:` 变体控制
- 主题切换按钮在 Navbar 上

---

## 5. 组件设计原则

- **纯展示组件** — 所有组件接收 props 或从数据文件读取数据，不包含业务逻辑
- **Section 容器统一** — 使用 `SectionWrapper` 统一管理标题、间距和入场动画
- **低耦合** — 组件之间不互相依赖，独立渲染
- **可复用** — 像 `Timeline` 组件同时处理工作和教育经历

## 6. 部署流程

```bash
# 1. 安装依赖
pnpm install

# 2. 本地开发
pnpm dev

# 3. 构建
pnpm build

# 4. 部署到 Vercel（连接 GitHub 仓库后自动部署）
#    或者手动部署：
pnpm vercel --prod
```
